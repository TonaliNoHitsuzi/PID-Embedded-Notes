# 第五章：嵌入式应用实战与跨领域拓展

---

## 5.0 从"调一个环"到"搭一套系统"

经过前四章，你已经掌握了 PID 的数学原理、C 语言实现和调参技巧。但真实的嵌入式系统很少只有一个 PID 环在孤军奋战。

一架无人机至少有 **4 个电机 × 3 轴姿态 = 12 个底层控制量**，加上位置外环，可能同时运行 20 多个 PID 实例。一个六轴机械臂的每个关节都需要位置伺服，还要协调运动轨迹。

本章将带你从"单环调试"跨越到"系统设计"，通过四个嵌入式核心案例，理解**串级 PID、前馈补偿、多环协同**的工程艺术。最后，我们跳出嵌入式，看看 PID 思想如何在其他领域开枝散叶。

---

## 5.1 直流电机：速度环与位置环的双环串级

这是嵌入式中最经典、最基础的 PID 应用场景，也是理解**串级控制**的最佳入口。

### 5.1.1 为什么单环不够？

假设你直接用位置 PID 控制电机：目标角度 180°，当前角度 0°，误差 180°。

- 如果 $K_p$ 小：电机慢悠悠转，10 秒才到位。
- 如果 $K_p$ 大：电机全速冲刺，冲到 180° 时惯性太大，刹不住，冲过头到 200°，然后来回震荡。

**问题根源**：位置环直接输出 PWM，但 PWM 本质上控制的是**力矩/速度**，不是位置。位置对 PWM 的响应存在"积分"关系（速度积分成位置），这让位置环变成了一个**积分型被控对象**，对 PID 极不友好。

### 5.1.2 串级结构：内环快，外环慢

**解决方案**：把系统拆成两个环。

```
位置目标 ──→[ 位置环 PID ]──→ 速度目标 ──→[ 速度环 PID ]──→ PWM ──→ 电机
                ↑                                    ↑
                └────── 编码器位置 ──────────────────┘
                       └────── 编码器速度 ───────────┘
```

- **内环（速度环）**：以编码器测得的转速为反馈，目标是快速、无静差地跟踪速度指令。采样周期通常 1~2 ms。
- **外环（位置环）**：以编码器测得的位置为反馈，输出的是"希望电机以多快的速度到达目标位置"。采样周期通常 5~10 ms。

### 5.1.3 为什么这样更好？

**物理直觉**：位置环不再直接指挥 PWM，而是给速度环"下订单"："我现在离目标还有 180°，请以 300 RPM 的速度接近。"速度环收到订单后，专业地控制 PWM 实现 300 RPM。当位置接近目标时，位置环自动降低速度指令（如 50 RPM → 10 RPM → 0），电机平滑地"滑"到目标点，不会冲过头。

**数学直觉**：速度环把电机这个"二阶振荡系统"改造成了一个"一阶惯性系统"（速度对 PWM 的响应近似一阶）。位置环面对的是一个更容易控制的被控对象。

### 5.1.4 参数设计原则：带宽分离

串级 PID 的黄金法则：**内环带宽 ≈ 5~10 倍外环带宽**。

- 先单独调速度环（位置环断开，直接给速度目标）。用阶跃速度指令，把速度环调得响应快、无超调。
- 再闭合位置环，调位置环参数。此时速度环已经是一个"快速执行器"，位置环只需要慢悠悠地发速度指令。

> **嵌入式注意**：两个环的采样周期可以不同。速度环 1ms 中断，位置环 5ms 中断（在速度环的每 5 次调用中，第 5 次才执行位置环计算）。这节省了 CPU 时间。

### 5.1.5 代码框架

```c
typedef struct {
    PID_Inc_t vel_pid;   // 内环：速度
    PID_Pos_t pos_pid;   // 外环：位置（位置环用位置式更直观）
    float target_pos;
    float current_pos;
    float current_vel;
} Motor_Cascade_t;

void Motor_Control(Motor_Cascade_t *motor, float pos, float vel)
{
    motor->current_pos = pos;
    motor->current_vel = vel;
    
    // 外环：位置环，输出为目标速度
    float target_vel = PID_Pos_Update(&motor->pos_pid, pos);
    
    // 限制目标速度（保护电机和机械结构）
    if (target_vel > VEL_MAX) target_vel = VEL_MAX;
    if (target_vel < -VEL_MAX) target_vel = -VEL_MAX;
    
    // 内环：速度环，输出为 PWM
    motor->vel_pid.setpoint = target_vel;
    float pwm = PID_Inc_Update(&motor->vel_pid, vel);
    
    TIM_SetPWM(pwm);
}

// 中断中调用（1ms）
void TIM_IRQHandler(void)
{
    static uint16_t pos_cnt = 0;
    float pos = Encoder_GetPosition();
    float vel = Encoder_GetVelocity(); // 通常通过位置差分计算
    
    // 位置环每 5ms 更新一次（降频）
    if (++pos_cnt >= 5) {
        pos_cnt = 0;
        motor.pos_pid.setpoint = motor.target_pos;
    }
    
    Motor_Control(&motor, pos, vel);
}
```

## 5.2 机械臂：关节控制与力/位混合

机械臂是 PID 从"一维"走向"多维"的桥梁。每个关节都是一个电机，但关节之间存在耦合。

### 5.2.1 单关节位置控制

每个关节独立使用**位置环 + 速度环**的串级结构，与 5.1 节相同。

**额外挑战**：

- **重力负载**：机械臂水平伸展时，肩关节承受最大重力矩；垂直时重力矩为零。这个重力矩是位置相关的已知扰动。
- **摩擦非线性**：关节减速器的静摩擦和动摩擦差异大，低速时可能出现"爬行"（stick-slip）。

**对策——前馈补偿**：

```c
// 根据当前关节角度，计算重力补偿力矩
float GravityCompensation(float angle)
{
    return ARM_MASS * GRAVITY * ARM_LENGTH * cosf(angle);
}

// 在速度环输出中加入前馈
float pwm = PID_Inc_Update(&vel_pid, vel) 
          + GravityCompensation(current_pos) * FEEDFORWARD_GAIN;
```

> **效果**：PID 只需处理"残余误差"（摩擦力、模型误差），重力由前馈"扛住"。这大幅降低了对 PID 增益的要求，系统更稳定。

### 5.2.2 力/位混合控制（简介）

机械臂在打磨、装配等任务中，需要同时控制**位置**和**接触力**。

- **位置控制模式**：自由空间中移动，PID 控制位置。
- **力控制模式**：接触工件后，切换到力 PID，控制末端执行器对工件的压力。

更高级的实现使用**阻抗控制**或**导纳控制**，但底层仍然依赖 PID 来跟踪期望的位置或力轨迹。

## 5.3 无人机：姿态环与位置环的嵌套

四旋翼无人机是串级 PID 最精彩的应用之一。它展示了如何在**三维空间**中层层拆解控制问题。

### 5.3.1 四旋翼的动力学直觉

四旋翼有 6 个自由度（位置 x,y,z + 姿态 roll,pitch,yaw），但只有 4 个控制输入（4 个电机的转速）。这意味着它是**欠驱动系统**——不能直接控制所有自由度，必须通过姿态间接控制位置。

**关键物理**：四旋翼的升力方向始终垂直于机身平面。如果机身前倾（pitch +），升力就有向前的水平分量，无人机向前加速。因此：

> **控制位置 → 先控制姿态 → 姿态控制好了，位置自然可控**

### 5.3.2 三层串级架构

典型的无人机飞控包含三层 PID（从外到内）：

```
位置外环 (x, y, z)
    │
    ▼ 输出：目标姿态角 + 目标油门
姿态中环 (roll, pitch, yaw)
    │
    ▼ 输出：目标角速度
角速度内环 (roll_rate, pitch_rate, yaw_rate)
    │
    ▼ 输出：电机差速（PWM）
```

**角速度环（最内环）**：

- 输入：IMU（陀螺仪）测得的三轴角速度。
- 目标：快速跟踪姿态环给出的角速度指令。
- 采样周期：1~2 ms（最高优先级）。
- 这是"最硬"的环，需要响应极快，抑制外部扰动（如阵风）。

**姿态环（中环）**：

- 输入：IMU（加速度计+陀螺仪融合）测得的 roll/pitch/yaw 角度。
- 目标：保持机身水平或跟踪期望姿态。
- 采样周期：2~5 ms。
- 输出：角速度指令（如 "请以 30°/s 的角速度向右倾斜"）。

**位置环（最外环）**：

- 输入：GPS/光流/视觉测得的位置。
- 目标：悬停在指定坐标，或按轨迹飞行。
- 采样周期：10~50 ms（GPS 更新慢）。
- 输出：期望姿态角（如 "前倾 5° 以向前飞行"）和油门。

### 5.3.3 参数设计：越内环越快

| 控制环 | 典型带宽 | 典型采样周期 | 关键要求 |
|--------|---------|-------------|---------|
| 角速度环 | 20~50 Hz | 1~2 ms | 快、稳，D 项常用 |
| 姿态环 | 5~10 Hz | 2~5 ms | 平滑，抑制超调 |
| 位置环 | 1~2 Hz | 10~50 ms | 允许一定滞后 |

**为什么内环必须有 D？**

无人机是二阶不稳定系统（类似倒立摆）。角速度环如果没有 D，仅靠 P 和 I，机身受到扰动后会持续振荡很久。D 提供"阻尼"，让机身像"有减震器"一样快速稳定。

### 5.3.4 代码结构示意

```c
typedef struct {
    PID_Inc_t rate_roll;   // 角速度环
    PID_Inc_t rate_pitch;
    PID_Inc_t rate_yaw;
    
    PID_Pos_t angle_roll;  // 姿态环
    PID_Pos_t angle_pitch;
    PID_Pos_t angle_yaw;
    
    PID_Pos_t pos_x;       // 位置环（简化示意）
    PID_Pos_t pos_y;
    PID_Pos_t pos_z;
} Drone_Controller_t;

void FlightControl_Update(Drone_Controller_t *fc, 
                          IMU_Data_t *imu, 
                          Setpoint_t *sp)
{
    // 1. 位置环（低频，如 10Hz）
    float target_roll  = PID_Pos_Update(&fc->pos_x, imu->pos_x);
    float target_pitch = PID_Pos_Update(&fc->pos_y, imu->pos_y);
    float target_throttle = PID_Pos_Update(&fc->pos_z, imu->pos_z);
    
    // 2. 姿态环（中频，如 200Hz）
    fc->angle_roll.setpoint  = target_roll;
    fc->angle_pitch.setpoint = target_pitch;
    fc->angle_yaw.setpoint   = sp->yaw_angle; // 航向通常直接给目标
    
    float rate_roll_sp  = PID_Pos_Update(&fc->angle_roll, imu->angle_roll);
    float rate_pitch_sp = PID_Pos_Update(&fc->angle_pitch, imu->angle_pitch);
    float rate_yaw_sp   = PID_Pos_Update(&fc->angle_yaw, imu->angle_yaw);
    
    // 3. 角速度环（高频，如 500Hz）
    fc->rate_roll.setpoint  = rate_roll_sp;
    fc->rate_pitch.setpoint = rate_pitch_sp;
    fc->rate_yaw.setpoint   = rate_yaw_sp;
    
    float roll_out  = PID_Inc_Update(&fc->rate_roll, imu->rate_roll);
    float pitch_out = PID_Inc_Update(&fc->rate_pitch, imu->rate_pitch);
    float yaw_out   = PID_Inc_Update(&fc->rate_yaw, imu->rate_yaw);
    
    // 4. 混控（Mixing）：把 roll/pitch/yaw/throttle 分配到 4 个电机
    Motor_Mix(roll_out, pitch_out, yaw_out, target_throttle);
}
```

## 5.4 倒立摆：不稳定系统的 PID 镇定

倒立摆是控制理论的经典教具，也是检验 PID 能力的"试金石"。

### 5.4.1 系统特性：开环不稳定

一根杆竖直立在运动的小车上。如果什么都不做，杆会自然倒下（稳定平衡点是杆朝下）。我们的目标是：**通过控制小车的左右移动，让杆保持竖直**。

从控制角度看，倒立摆是：

- **开环不稳定**：没有控制时，杆必然倒下。
- **非线性**：角度大时和小时的动力学完全不同。
- **欠驱动**：只能控制小车位置，不能直接控制力矩。

### 5.4.2 控制策略：角度环 + 位置环

直觉上，当杆向右倒时，小车应该向右加速，用惯性把杆"推"回竖直。这需要一个**角度 PID**：

```c
// 角度环：目标是 0°（竖直）
float target_cart_pos = PID_Pos_Update(&angle_pid, imu->pendulum_angle);
```

但仅有角度环不够。小车会为了追杆而不断向一个方向跑，最终撞到轨道尽头。因此需要外层的**位置环**：

```c
// 位置环：限制小车在轨道中心附近
float limited_pos = PID_Pos_Update(&cart_pos_pid, encoder->cart_position);
angle_pid.setpoint = limited_pos; // 角度环的目标不是严格0，而是允许小偏移
```

更常见的实现是**角度环 + 速度环**：

- 角度环输出"目标小车速度"（杆倒向右边，小车应向右以多大速度移动）。
- 速度环控制电机实现该速度。

### 5.4.3 为什么纯 PID 能搞定？

严格来说，线性化的倒立摆模型需要状态反馈（LQR）才能达到最优。但在**小角度范围**（如 ±15°）内，PID 完全可以镇定系统，因为：

- 小角度近似下，系统动力学接近线性。
- PID 的 D 项提供了关键的"阻尼"，防止杆来回摇摆。
- 串级结构把复杂问题分解成了两个简单的跟踪问题。

> **嵌入式挑战**：角度传感器（编码器或 IMU）的安装位置、采样延迟、小车传动间隙都会影响稳定性。实际调试中，D 项的滤波和采样周期的严格固定至关重要。

## 5.5 跨领域拓展：PID 思想的跨界旅行

PID 不仅属于嵌入式。它的核心思想——**基于误差的反馈、比例-积分-微分的组合**——已经渗透到许多看似无关的领域。

### 5.5.1 化工过程控制：温度、压力、流量

这是 PID 的"老家"。反应釜温度控制：

- **P**：温差大时加大加热功率。
- **I**：消除长期温度偏差（环境温度变化、物料热容变化）。
- **D**：当温度上升过快时提前减少加热，防止过冲（保护催化剂）。

与嵌入式的区别：

- 采样周期慢（秒级甚至分钟级）。
- 被控对象大滞后（传感器离加热区远），D 项使用需谨慎。
- 通常用 PLC 实现，有现成的 PID 功能块。

### 5.5.2 自动驾驶：横向路径跟踪

自动驾驶车辆需要让方向盘转动，使车辆始终沿着车道中心线行驶。

**横向误差** = 车辆当前位置到车道中心线的垂直距离。

**PID 控制**：

- **P**：偏离中心线越远，方向盘打得越急。
- **I**：消除长期偏驶（如横风、路面倾斜导致的持续跑偏）。
- **D**：当车辆快速接近中心线时（如从左侧大偏移回正），提前回正方向盘，防止"蛇形"摆动。

> **与嵌入式 PID 的区别**：车辆动力学是强非线性的（轮胎侧偏特性），高速和低速时同样的方向盘角度效果完全不同。因此自动驾驶通常使用**增益调度 PID**（不同速度区间用不同参数）或更高级的模型预测控制（MPC）。

### 5.5.3 机器人 SLAM：运动控制

SLAM（同步定位与建图）算法输出的是期望速度指令，但底盘执行时存在打滑、电机不一致等问题。

底层仍然是一个**速度 PID**：

- 左右轮独立速度环，确保直行时不跑偏。
- 外环是航向角 PID（用 IMU 或里程计反馈），确保 SLAM 给出的方向指令被精确执行。

### 5.5.4 计算领域：流量控制与负载均衡

这是 PID 最"抽象"的应用。

**TCP 拥塞控制（类 PID）**：

- **P**：当前网络拥塞（丢包/延迟大）→ 成比例减小发送窗口。
- **I**：长期带宽探测 → 缓慢增加窗口，直到再次拥塞（AIMD 中的 AI）。
- **D**：预测拥塞（如 RTT 变长趋势）→ 提前减速。

**服务器负载均衡**：

- 监控 CPU 使用率（误差 = 目标负载 - 实际负载）。
- **P**：负载高时多分配任务到其他服务器。
- **I**：长期负载不均时调整权重。
- **D**：负载快速上升时提前扩容（云服务的自动伸缩）。

> **本质洞察**：任何存在"目标值-实际值"差异、需要动态调节的闭环系统，都可以从 PID 思想中获益。PID 是一种**元控制哲学**。

## 5.6 本章小结

| 应用场景 | 控制结构 | 关键技巧 |
|---------|---------|---------|
| **直流电机** | 速度环（内）+ 位置环（外） | 带宽分离，内环快 5~10 倍 |
| **机械臂** | 位置/速度串级 + 重力前馈 | 前馈补偿已知扰动，PID 处理残余 |
| **无人机** | 角速度→姿态→位置 三层串级 | 内环必须有 D 提供阻尼，采样周期分层 |
| **倒立摆** | 角度环 + 速度/位置环 | 小角度线性化，D 项防振荡 |
| **化工/自动驾驶** | 单环或串级 PID | 大滞后慎用 D，非线性用增益调度 |
| **计算领域** | 类 PID 思想 | 抽象为误差反馈，比例积分微分隐喻 |

## 5.7 下章预告

PID 虽然强大，但并非万能。它假设系统是**线性、单输入单输出、时不变**的。当面对以下挑战时，传统 PID 开始力不从心：

- 机械臂高速运动时，动力学剧烈变化（**时变系统**）。
- 化工反应釜在不同温度区间的特性完全不同（**非线性**）。
- 多旋翼飞行器存在轴间耦合（**多变量 MIMO**）。

第六章将探讨 PID 的边界，以及工业界和学术界如何改进它：

- **模糊 PID**：用人类经验替代精确数学。
- **自适应 PID**：参数随工况自动调整。
- **神经网络 PID**：用深度学习替代人工调参。
- **分数阶 PID**：打破整数阶微积分的束缚。
- **串级、前馈、解耦**：用结构改进而非算法改进。
