# 第四章：参数整定与调试技巧

---

## 4.0 调参不是"玄学"

很多嵌入式工程师对 PID 调参有一种恐惧——"先调大 P，不行再加 I，再抖就加 D，还是抖就减 P……" 这种**盲人摸象**式的调参，既浪费时间，又可能烧毁电机。

实际上，PID 调参是一门**有方法论、有流程、可复现**的工程技艺。本章将给你一套**从仿真到上板、从离线到在线**的完整调参武器库。

---

## 4.1 经典整定法：Ziegler-Nichols

Ziegler-Nichols（Z-N）整定法是 1942 年提出的工程经验方法，至今仍是最实用的"第一组参数"来源。它有两种形式：

### 4.1.1 临界比例度法（闭环法）

**核心思想**：先只用 P 控制，逐渐增大 $K_p$ 直到系统产生**等幅振荡**（既不收敛也不发散），记录此时的 $K_p$ 和振荡周期 $T_u$。

**在 MCU 上的实操步骤**：

1. 将 $K_i = 0, K_d = 0$，设定一个中等 $K_p$（如 1.0）。
2. 给系统一个阶跃目标值（如电机从 0 到 3000 RPM）。
3. 观察输出响应：
   - 如果收敛且无振荡 → $K_p$ 偏小，增大 50%。
   - 如果衰减振荡 → 接近临界，继续微调。
   - 如果等幅振荡（像正弦波一样一直晃）→ **记录当前 $K_p = K_u$**，用示波器或串口打印测量振荡周期 $T_u$。
4. 按 Z-N 表计算 PID 参数。

**Z-N 临界比例度整定表**：

| 控制器类型 | $K_p$ | $T_i$ | $T_d$ | 离散化后（ $K_i = K_p T/T_i$, $K_d = K_p T_d/T$）|
|-----------|-------|-------|-------|-----------------------------------------------|
| P | $0.5 K_u$ | — | — | — |
| PI | $0.45 K_u$ | $T_u / 1.2$ | — | — |
| PID | $0.6 K_u$ | $T_u / 2$ | $T_u / 8$ | $K_i = 1.2 K_p T/T_u$, $K_d = 0.075 K_p T_u / T$ |

> **嵌入式技巧**：在 MCU 上判断"等幅振荡"不需要示波器。你可以通过串口每隔 10ms 输出实际值，用 Python 或 Excel 画图观察。或者更简单：如果连续 10 个波峰幅度相差 < 5%，就认为是等幅振荡。

**临界法的局限性**：

- 有些系统不允许振荡（如化工反应釜、机械臂可能撞限位）。
- 找到精确临界点需要耐心，且 $K_u$ 可能让系统短暂失控。

### 4.1.2 阶跃响应法（开环法）

如果系统不允许闭环振荡，可以用开环法。给被控对象一个阶跃输入（如 PWM 从 0 到 50%），记录输出的响应曲线。

典型的响应曲线呈 **S 型**（一阶惯性加纯滞后），找到两个特征参数：

- **L**：滞后时间（阶跃开始到曲线明显拐弯的时间）
- **T**：时间常数（曲线从 0 上升到 63.2% 稳态值的时间）

**Z-N 阶跃响应整定表**：

| 控制器类型 | $K_p$ | $T_i$ | $T_d$ |
|-----------|-------|-------|-------|
| P | $T / L$ | — | — |
| PI | $0.9 T / L$ | $3.33 L$ | — |
| PID | $1.2 T / L$ | $2 L$ | $0.5 L$ |

**在 MCU 上的实操**：

- 直接输出固定 PWM（如 50%），用编码器/ADC 记录响应。
- 通过串口把数据发到上位机，拟合出 L 和 T。
- 适合温度、液位等允许开环测试的系统。

### 4.1.3 Z-N 参数的后续修正

Z-N 法给出的参数通常**响应快但超调大**（约 20%~25%）。在实际嵌入式系统中，往往需要"保守化"：

- 将 $K_p$ 乘以 0.5~0.7。
- 将 $T_i$ 乘以 1.5~2（让积分更慢）。
- 如果超调仍大，适当增大 $T_d$。

## 4.2 调参顺序：先 P 后 I 再 D

无论用哪种方法，上板后的微调都遵循这个口诀。理解每一步的物理意义，才能知道该往哪个方向调。

### 4.2.1 第一步：调 P（比例）

**目标**：让系统"有力量"地响应，允许有静差，但不剧烈振荡。

**操作**：

- $K_i = 0, K_d = 0$。
- 从小到大逐步增大 $K_p$。
- 给阶跃目标值，观察响应。

**判断标准**：

- $K_p$ 太小：响应 sluggish，像"没吃饱"一样无力。
- $K_p$ 适中：快速上升，有静差（如目标 3000，稳在 2800），基本不振荡。
- $K_p$ 太大：快速上升，但超调严重，衰减振荡。

**停止条件**：找到"刚好开始振荡"的 $K_p$，然后回退到 60%~70%。

### 4.2.2 第二步：调 I（积分）

**目标**：消除静差，不引入新的振荡。

**操作**：

- 保持上一步的 $K_p$。
- 从小到大逐步增大 $K_i$。
- 观察静差是否消除，以及是否出现超调。

**判断标准**：

- $K_i$ 太小：静差消除慢，甚至一直存在。
- $K_i$ 适中：静差消除，超调 < 10%。
- $K_i$ 太大：静差消除快，但出现"积分滞后"导致的振荡，恢复时间长。

**停止条件**：静差消除，且超调可接受。

### 4.2.3 第三步：调 D（微分）

**目标**：抑制超调，加速收敛。

**操作**：

- 保持 $K_p, K_i$。
- 从小到大逐步增大 $K_d$。
- 观察超调量和调节时间。

**判断标准**：

- $K_d$ 太小：超调无明显改善。
- $K_d$ 适中：超调减小，曲线更"圆润"，更快稳定。
- $K_d$ 太大：系统变"僵硬"，响应迟钝，执行器出现高频抖动（噪声被放大）。

**停止条件**：超调最小，且无抖动。

> **嵌入式口诀**：P 管快慢，I 管静差，D 管超调。先让系统"有力气"，再让系统"不偷懒"，最后让系统"不冲动"。

## 4.3 嵌入式在线调参策略

在实验室里，你可以把 MCU 连到电脑，用 MATLAB 调参。但在现场（如无人机试飞、机器人调试），你需要**不烧录程序就能改参数**的能力。

### 4.3.1 串口调参（最推荐）

通过串口发送指令，实时修改 PID 参数。

**通信协议设计**（简单可靠）：

```
格式：CMD=VALUE\r\n
示例：
Kp=2.5\n
Ki=0.8\n
Kd=0.05\n
Set=3000\n
```

**MCU 端解析代码**：

```c
// 在串口中断中接收
void USART1_IRQHandler(void)
{
    static char buf[32];
    static uint8_t idx = 0;
    
    if (USART1->SR & USART_SR_RXNE) {
        char c = USART1->DR;
        if (c == '\n' || c == '\r') {
            buf[idx] = '\0';
            PID_ParseCommand(buf);
            idx = 0;
        } else if (idx < 31) {
            buf[idx++] = c;
        }
    }
}

void PID_ParseCommand(const char *cmd)
{
    float val;
    if (sscanf(cmd, "Kp=%f", &val) == 1) {
        motor_pid.Kp = val;
        printf("Kp set to %.3f\r\n", val);
    } else if (sscanf(cmd, "Ki=%f", &val) == 1) {
        motor_pid.Ki = val;
        printf("Ki set to %.3f\r\n", val);
    } else if (sscanf(cmd, "Kd=%f", &val) == 1) {
        motor_pid.Kd = val;
        printf("Kd set to %.3f\r\n", val);
    } else if (sscanf(cmd, "Set=%f", &val) == 1) {
        motor_pid.setpoint = val;
        printf("Setpoint set to %.1f\r\n", val);
    }
}
```

**上位机配合**：

- 最简单：用串口助手手动发送 `Kp=2.5`。
- 进阶：用 Python 写一个小脚本，自动扫描参数空间，记录响应曲线。

### 4.3.2 电位器/按键调参（无电脑场景）

在无人机试飞、野外机器人调试时，可能没有电脑。

**电位器方案**：

- 用 ADC 读取两个电位器（如摇杆），分别映射到 $K_p$ 和 $K_i$。
- 实时显示在 OLED 屏幕上。
- 按下按键确认，将参数写入 Flash。

**按键方案**：

- 短按： $K_p$ +0.1
- 长按： $K_p$ -0.1
- 组合键切换调节对象（P/I/D/Setpoint）

### 4.3.3 参数掉电保存

调好的参数不能每次上电都重调。用 MCU 的 Flash 模拟 EEPROM，或直接用 EEPROM：

```c
#include <stm32f1xx_hal_flash.h>

#define PARAM_ADDR 0x0800FC00  // Flash 最后一页（需根据芯片调整）

void PID_SaveParams(PID_Inc_t *pid)
{
    HAL_FLASH_Unlock();
    FLASH_EraseInitTypeDef erase;
    erase.TypeErase = FLASH_TYPEERASE_PAGES;
    erase.PageAddress = PARAM_ADDR;
    erase.NbPages = 1;
    uint32_t error = 0;
    HAL_FLASHEx_Erase(&erase, &error);
    
    // 写入参数（按 float 的 32 位原始值写入）
    HAL_FLASH_Program(FLASH_TYPEPROGRAM_WORD, PARAM_ADDR, 
                      *(uint32_t*)&pid->Kp);
    HAL_FLASH_Program(FLASH_TYPEPROGRAM_WORD, PARAM_ADDR + 4, 
                      *(uint32_t*)&pid->Ki);
    HAL_FLASH_Program(FLASH_TYPEPROGRAM_WORD, PARAM_ADDR + 8, 
                      *(uint32_t*)&pid->Kd);
    HAL_FLASH_Lock();
}

void PID_LoadParams(PID_Inc_t *pid)
{
    pid->Kp = *(float*)PARAM_ADDR;
    pid->Ki = *(float*)(PARAM_ADDR + 4);
    pid->Kd = *(float*)(PARAM_ADDR + 8);
    
    // 合法性检查：防止第一次上电读到全0xFF
    if (pid->Kp < 0 || pid->Kp > 1000) pid->Kp = 1.0f;
    if (pid->Ki < 0 || pid->Ki > 1000) pid->Ki = 0.0f;
    if (pid->Kd < 0 || pid->Kd > 1000) pid->Kd = 0.0f;
}
```

## 4.4 问题诊断手册：症状、原因、对策

调参时遇到问题，不要"乱调一气"。用这张表按图索骥：

| 症状 | 可能原因 | 诊断方法 | 对策 |
|------|---------|---------|------|
| **系统完全不响应** | $K_p$ 太小，或执行器未使能 | 检查 PWM/输出是否有变化 | 增大 $K_p$ 到 10 倍试试；检查硬件 |
| **响应极慢，像没劲** | $K_p$ 太小，或采样周期太长 | 观察上升时间是否 > 5s | 增大 $K_p$；缩短 $T$ |
| **有静差，永远差一截** | $K_i = 0$ 或太小 | 观察稳态值是否始终低于目标 | 引入或增大 $K_i$ |
| **等幅振荡** | $K_p$ 接近临界值，或 $K_i$ 太大 | 波形像正弦，幅度不增不减 | 减小 $K_p$ 30%；或减小 $K_i$ |
| **衰减振荡，恢复慢** | $K_p$ 稍大，或 $K_d$ 不够 | 振荡几次后收敛 | 适当减小 $K_p$ 或增大 $K_d$ |
| **超调大，冲过头** | $K_p$ 太大，或 $K_d$ 太小 | 第一次峰值远超目标 | 减小 $K_p$ 或增大 $K_d$ |
| **高频抖动，嗡嗡响** | $K_d$ 太大，或传感器噪声 | 输出在目标值附近高频振动 | 减小 $K_d$；加微分滤波；检查传感器 |
| **大误差时正常，小误差时振荡** | 积分饱和后释放 | 观察是否在接近目标时突然冲过头 | 加积分限幅；用条件积分 |
| **设定值突变时执行器猛冲** | 微分冲击（Derivative Kick） | 只在设定值改变时发生 | 改用微分先行 |
| **系统偶尔发疯，输出满幅** | 数值溢出，或传感器断线 | 检查 error 是否突然变成极大值 | 加 error 限幅；传感器故障检测 |

### 4.4.1 诊断流程图（文字版）

```
开始调参
  │
  ▼
系统有响应吗？
  ├─ 否 → 检查硬件/Kp是否太小/执行器使能
  ▼
有静差吗？
  ├─ 是 → 引入 Ki，从小到大
  ▼
有振荡吗？
  ├─ 是 → 振荡是等幅还是衰减？
  │         ├─ 等幅 → Kp 太大，减小 30%
  │         └─ 衰减 → 适当减小 Kp 或引入 Kd
  ▼
超调大吗？
  ├─ 是 → 引入 Kd，从小到大
  ▼
执行器抖动吗？
  ├─ 是 → Kd 太大或传感器噪声，减小 Kd 或加滤波
  ▼
完成
```

## 4.5 仿真预调参：先电脑，后上板

在 MCU 上烧程序、看现象、改参数、再烧录——这个循环太慢。更聪明的做法是：**先在电脑上仿真，找到大致安全的参数范围，再上板微调。**

### 4.5.1 为什么仿真能救命？

- 仿真中 $K_p$ 调得再大也不会烧电机。
- 可以批量扫描参数空间，找到最优组合。
- 能清晰看到"如果去掉 D 会怎样"的对比。

### 4.5.2 Python 仿真示例

以下是一个完整的离散 PID 仿真脚本，模拟一个典型的**电机二阶系统**：

```python
import numpy as np
import matplotlib.pyplot as plt

# 系统参数（模拟一个电机：二阶系统）
dt = 0.001  # 采样周期 1ms
T = np.arange(0, 2, dt)  # 仿真 2 秒
setpoint = np.ones_like(T) * 1000  # 目标值 1000 RPM

# 被控对象：二阶离散传递函数近似
# y[k] = 1.8*y[k-1] - 0.9*y[k-2] + 0.05*u[k-1] + 0.01*u[k-2]
def plant(u_hist, y_hist):
    if len(y_hist) < 2:
        return 0
    return 1.8*y_hist[-1] - 0.9*y_hist[-2] + 0.05*u_hist[-1] + 0.01*u_hist[-2]

def simulate_pid(Kp, Ki, Kd):
    y = [0, 0]      # 输出历史
    u = [0, 0]      # 控制量历史
    e_hist = [0, 0] # 误差历史
    
    integral = 0
    output = []
    
    for k in range(len(T)):
        e = setpoint[k] - y[-1]
        integral += e * dt
        
        # 位置式 PID
        p = Kp * e
        i = Ki * integral
        d = Kd * (e - e_hist[-1]) / dt
        
        uk = p + i + d
        uk = max(0, min(uk, 2000))  # 输出限幅
        
        y_new = plant(u, y)
        y.append(y_new)
        u.append(uk)
        e_hist.append(e)
        output.append(y_new)
    
    return np.array(output)

# 对比不同参数
y1 = simulate_pid(Kp=2.0, Ki=0, Kd=0)      # 纯 P
y2 = simulate_pid(Kp=2.0, Ki=50, Kd=0)     # PI
y3 = simulate_pid(Kp=2.0, Ki=50, Kd=0.001) # PID

plt.figure(figsize=(10, 6))
plt.plot(T, y1, label='P only', alpha=0.7)
plt.plot(T, y2, label='PI', alpha=0.7)
plt.plot(T, y3, label='PID', alpha=0.7)
plt.axhline(1000, color='r', linestyle='--', label='Setpoint')
plt.xlabel('Time (s)')
plt.ylabel('Output (RPM)')
plt.legend()
plt.title('PID Step Response Comparison')
plt.grid(True)
plt.show()
```

**运行结果解读**：

- **纯 P**：快速上升，但稳态值约 800（静差 200）。
- **PI**：静差消除，但超调约 15%。
- **PID**：超调几乎消失，快速收敛。

> **嵌入式实践**：在电脑上跑通仿真后，把 $K_p, K_i, K_d$ 作为初始值烧进 MCU，通常只需微调即可。

## 4.6 进阶调参技巧

### 4.6.1 变积分策略

误差很大时，积分不应全力工作（否则容易饱和）；误差小时，积分应全力消除静差。

```c
float error_abs = fabs(pid->error);
float ki_factor = (error_abs > 100) ? 0.3f : 1.0f;  // 大误差时积分减弱
pid->integral += pid->error * ki_factor;
```

### 4.6.2 不完全微分（滤波微分）

在 D 通道加一个一阶低通滤波，牺牲一点相位超前，换取噪声抑制：

```c
#define D_FILTER_COEF 0.3f

float derivative_raw = pid->error - pid->last_error;
pid->derivative = D_FILTER_COEF * derivative_raw 
                + (1.0f - D_FILTER_COEF) * pid->derivative;
```

### 4.6.3 前馈-反馈复合控制

对于已知扰动（如机械臂的重力负载、无人机的已知风力），可以前馈补偿，减轻 PID 负担：

$$
u = u_{PID} + u_{feedforward}
$$

例如，机械臂关节已知重力矩为 $G(\theta)$，直接把这个力矩加到输出上，PID 只需处理残余误差。

## 4.7 本章小结

| 要点 | 核心内容 |
|------|---------|
| **Z-N 临界法** | 只用 P，增大到等幅振荡，记录 $K_u, T_u$，查表得初始参数 |
| **Z-N 开环法** | 阶跃响应测 L 和 T，适合不允许振荡的系统 |
| **调参顺序** | 先 P（有力量），再 I（消静差），后 D（抑超调） |
| **在线调参** | 串口指令最灵活，电位器/按键适合野外，参数存 Flash |
| **问题诊断** | 按症状查表：不振荡→减 P，有静差→加 I，超调大→加 D，抖动→减 D |
| **仿真预调** | Python/MATLAB 先验证，上板后微调，避免烧硬件 |
