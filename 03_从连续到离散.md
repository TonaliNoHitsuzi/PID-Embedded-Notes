# 第三章：从连续到离散——嵌入式实现路径

---

## 3.0 从公式到代码的鸿沟

第二章结尾，我们得到了优雅的连续 PID 公式：

$$
u(t) = K_p \cdot e(t) + K_i \int e(t) \, dt + K_d \frac{de(t)}{dt}
$$

但当你打开 STM32 的 Keil 工程，面对的是一个**每 1ms 触发一次**的定时器中断。这里没有积分符号，没有微分算子，只有 `int` 和 `float`，只有加减乘除。

本章的任务，就是把数学公式"翻译"成能在单片机上稳定运行的 C 代码。这不仅是符号替换，更是一场**工程化的妥协艺术**——你要在精度、速度、稳定性、内存之间找到平衡。

## 3.1 离散化：把微积分变成加减乘除

### 3.1.1 后向差分法

嵌入式系统是离散时间系统。设采样周期为 $T$（单位：秒），当前时刻为 $k$，则：

- **积分** → **累加**（矩形法近似）：

$$
\int_0^t e(\tau) \, d\tau \approx T \sum_{i=0}^{k} e(i)
$$

- **微分** → **差分**：

$$
\frac{de(t)}{dt} \approx \frac{e(k) - e(k-1)}{T}
$$

代入连续 PID 公式，得到**位置式离散 PID**：

$$
u(k) = K_p \cdot e(k) + K_i \cdot T \sum_{i=0}^{k} e(i) + K_d \cdot \frac{e(k) - e(k-1)}{T}
$$

为了简化，通常直接把 $T$ 合并进参数中：

$$
u(k) = K_p \cdot e(k) + K_i' \sum_{i=0}^{k} e(i) + K_d' \cdot [e(k) - e(k-1)]
$$

其中：

- $K_i' = K_i \cdot T$（离散积分系数）
- $K_d' = K_d / T$（离散微分系数）

> **工程直觉**：采样周期 $T$ 越小， $K_i'$ 越小（积分变慢）， $K_d'$ 越大（微分变敏感）。这意味着：**同样的 $K_p, K_i, K_d$，在不同采样周期下表现完全不同。** 调参时必须固定 $T$。

## 3.2 位置式 vs 增量式：两种实现哲学

离散 PID 有两种经典实现形式，选择哪一个，直接影响你的代码结构和系统可靠性。

### 3.2.1 位置式 PID（Positional PID）

输出的是**控制量的绝对值**（如 PWM 占空比 0~1000）。

$$
u(k) = K_p \cdot e(k) + K_i \cdot \sum e(i) + K_d \cdot [e(k) - e(k-1)]
$$

**特点**：

- 直接输出控制量，直观。
- 需要保存积分累积值。
- 积分饱和风险高。
- 适合需要绝对位置控制的场景（如舵机角度、加热功率）。

### 3.2.2 增量式 PID（Incremental PID）

输出的是**控制量的变化量** $\Delta u(k)$，再叠加到上一时刻输出上。

$$
\Delta u(k) = K_p \cdot [e(k) - e(k-1)] + K_i \cdot e(k) + K_d \cdot [e(k) - 2e(k-1) + e(k-2)]
$$

$$
u(k) = u(k-1) + \Delta u(k)
$$

**特点**：

- 输出是增量，天然限幅方便。
- 积分项隐含在累加过程中，抗饱和更优雅。
- 故障时输出不会突变（安全）。
- 适合执行器是"调速"类（如电机 PWM、阀门开度变化）。

### 3.2.3 选型对比

| 维度 | 位置式 | 增量式 |
|------|--------|--------|
| **输出含义** | 绝对控制量 | 控制量变化 |
| **积分处理** | 显式累加，需防饱和 | 隐式累加，天然限幅 |
| **故障安全** | 失控时输出可能满幅 | 失控时输出保持原值 |
| **代码复杂度** | 简单 | 稍复杂（多记一个历史误差） |
| **适用场景** | 温度控制、位置伺服 | 电机调速、流量调节 |
| **嵌入式推荐** | 需配合抗饱和算法 | 优先推荐，尤其电机类 |

> **嵌入式建议**：如果你在做电机控制、无人机姿态环，**优先用增量式**。它的故障安全特性（程序跑飞时输出不会突变）和抗饱和特性，在嵌入式这种"随时可能踩坑"的环境里更可靠。

## 3.3 嵌入式 C 代码实现

以下代码遵循**可移植、模块化、防御性编程**原则，可直接用于 STM32、ESP32、Arduino 等平台。

### 3.3.1 增量式 PID（推荐用于电机/调速类）

```c
/* pid.h */
#ifndef __PID_H
#define __PID_H

#include <stdint.h>

typedef struct {
    float Kp;           // 比例系数
    float Ki;           // 积分系数（已含采样周期T）
    float Kd;           // 微分系数（已含采样周期T）
    
    float setpoint;     // 目标值
    float output;       // 当前输出（绝对值）
    float output_min;   // 输出下限
    float output_max;   // 输出上限
    
    float error;        // 当前误差 e(k)
    float last_error;   // 上一时刻误差 e(k-1)
    float prev_error;   // 上两时刻误差 e(k-2)
    
    float delta_out_max; // 输出变化率限制（可选，防冲击）
} PID_Inc_t;

void PID_Inc_Init(PID_Inc_t *pid, float kp, float ki, float kd);
float PID_Inc_Update(PID_Inc_t *pid, float measurement);

#endif
```

```c
/* pid.c */
#include "pid.h"

void PID_Inc_Init(PID_Inc_t *pid, float kp, float ki, float kd)
{
    pid->Kp = kp;
    pid->Ki = ki;
    pid->Kd = kd;
    pid->setpoint = 0.0f;
    pid->output = 0.0f;
    pid->error = 0.0f;
    pid->last_error = 0.0f;
    pid->prev_error = 0.0f;
    pid->delta_out_max = 0.0f; // 0表示不限制
}

float PID_Inc_Update(PID_Inc_t *pid, float measurement)
{
    /* 1. 计算当前误差 */
    pid->error = pid->setpoint - measurement;
    
    /* 2. 计算增量式三项 */
    float delta_error  = pid->error - pid->last_error;           // e(k) - e(k-1)
    float delta2_error = pid->error - 2.0f*pid->last_error 
                       + pid->prev_error;                        // e(k) - 2e(k-1) + e(k-2)
    
    float delta_output = pid->Kp * delta_error 
                       + pid->Ki * pid->error 
                       + pid->Kd * delta2_error;
    
    /* 3. 限制输出变化率（可选，防止设定值突变时执行器冲击） */
    if (pid->delta_out_max > 0.0f) {
        if (delta_output > pid->delta_out_max) 
            delta_output = pid->delta_out_max;
        else if (delta_output < -pid->delta_out_max) 
            delta_output = -pid->delta_out_max;
    }
    
    /* 4. 累加得到绝对输出 */
    pid->output += delta_output;
    
    /* 5. 输出限幅（硬限幅） */
    if (pid->output > pid->output_max) 
        pid->output = pid->output_max;
    else if (pid->output < pid->output_min) 
        pid->output = pid->output_min;
    
    /* 6. 更新历史误差 */
    pid->prev_error = pid->last_error;
    pid->last_error = pid->error;
    
    return pid->output;
}
```

### 3.3.2 位置式 PID（推荐用于温度/位置类）

```c
typedef struct {
    float Kp, Ki, Kd;
    float setpoint;
    float output;
    float output_min, output_max;
    
    float error;
    float last_error;
    float integral;
    float integral_max;  // 积分上限（抗饱和）
} PID_Pos_t;

float PID_Pos_Update(PID_Pos_t *pid, float measurement)
{
    pid->error = pid->setpoint - measurement;
    
    /* 积分项：带限幅累加 */
    pid->integral += pid->error;
    if (pid->integral > pid->integral_max) 
        pid->integral = pid->integral_max;
    else if (pid->integral < -pid->integral_max) 
        pid->integral = -pid->integral_max;
    
    /* 计算输出 */
    pid->output = pid->Kp * pid->error 
                + pid->Ki * pid->integral 
                + pid->Kd * (pid->error - pid->last_error);
    
    /* 输出限幅 */
    if (pid->output > pid->output_max) 
        pid->output = pid->output_max;
    else if (pid->output < pid->output_min) 
        pid->output = pid->output_min;
    
    pid->last_error = pid->error;
    return pid->output;
}
```

### 3.3.3 使用示例（STM32 定时器中断中）

```c
PID_Inc_t motor_pid;

void TIM2_IRQHandler(void)
{
    if (TIM2->SR & TIM_SR_UIF) {
        TIM2->SR &= ~TIM_SR_UIF;  // 清中断标志
        
        float speed = Encoder_GetSpeed();  // 读取编码器转速
        float pwm = PID_Inc_Update(&motor_pid, speed);
        TIM1->CCR1 = (uint16_t)pwm;        // 输出PWM
    }
}

int main(void)
{
    // 初始化：Kp=2.0, Ki=0.5, Kd=0.1（这些值需根据实际系统调参）
    PID_Inc_Init(&motor_pid, 2.0f, 0.5f, 0.1f);
    motor_pid.setpoint = 3000.0f;  // 目标 3000 RPM
    motor_pid.output_min = 0.0f;
    motor_pid.output_max = 1000.0f; // 对应 PWM 满占空比
    
    // ... 初始化定时器、编码器、PWM ...
    while(1) {
        // 主循环处理通信、显示等
    }
}
```

## 3.4 采样周期：多快才算够快？

### 3.4.1 香农定理的工程直觉

奈奎斯特-香农采样定理说：采样频率必须大于信号最高频率的 2 倍，才能无失真重建。

**但在控制工程中，"2 倍"远远不够。** 控制环需要在误差刚出现时就做出反应，而不是等半个周期后再说。工程经验：

> **采样频率 ≥ 系统带宽的 10~20 倍**，即 $T \leq \frac{1}{10f_{BW}}$

### 3.4.2 不同系统的经验值

| 被控对象 | 典型带宽 | 推荐采样周期 $T$ | 推荐中断频率 |
|---------|---------|-----------------|-------------|
| 温度控制 | 0.01~0.1 Hz | 100 ms ~ 1 s | 1~10 Hz |
| 直流电机转速 | 10~50 Hz | 1~5 ms | 200~1000 Hz |
| 伺服电机位置 | 50~200 Hz | 0.5~2 ms | 500~2000 Hz |
| 无人机姿态 | 50~100 Hz | 2~5 ms | 200~500 Hz |
| 机械臂力控 | 100~500 Hz | 0.2~1 ms | 1~5 kHz |

### 3.4.3 采样周期与参数的关系

如果你已经通过仿真（MATLAB）得到了连续域参数 $K_p, T_i, T_d$，转换为离散参数：

$$
K_i = K_p \cdot \frac{T}{T_i}, \quad K_d = K_p \cdot \frac{T_d}{T}
$$

> **关键陷阱**：如果你把采样周期从 1ms 改成 5ms， $K_i$ 会变大 5 倍， $K_d$ 会变小 5 倍。系统可能立刻振荡或失稳。**调参时固定 $T$，改 $T$ 时重调参数。**

## 3.5 工程陷阱与防御策略

### 3.5.1 积分饱和（Integral Windup）

**现象**：电机堵转时，PWM 已经 100% 了，但转速上不去。积分项继续疯狂累加，等到堵转解除，电机不会立刻减速，而是带着巨大的"积分惯性"冲过头。

**防御——积分限幅**：

```c
if (pid->integral > pid->integral_max) 
    pid->integral = pid->integral_max;
```

**防御——积分分离**（大误差时不积分）：

```c
if (fabs(pid->error) < ERROR_THRESHOLD) {
    pid->integral += pid->error;  // 小误差才积分
}
```

**防御——条件积分（Clamping）**：

当输出已经饱和时，停止积分。这是最优雅的方法：

```c
// 仅在输出未饱和且误差方向与输出方向一致时积分
if ((pid->output < pid->output_max && pid->error > 0) ||
    (pid->output > pid->output_min && pid->error < 0)) {
    pid->integral += pid->error;
}
```

### 3.5.2 微分冲击（Derivative Kick）

**现象**：你突然把目标值从 0 改到 1000（阶跃输入）。误差瞬间从 0 变成 1000，微分项 $K_d \cdot (1000 - 0)$ 产生巨大冲击，执行器猛地一抖。

**防御——微分先行（Derivative on Measurement）**：

不对误差微分，而对**测量值**微分：

$$
u_D(k) = -K_d \cdot [y(k) - y(k-1)]
$$

因为 $e = r - y$，当 $r$ 突变时， $de = dr - dy$。如果 $r$ 是阶跃， $dr$ 无穷大。但对 $y$ 微分， $r$ 的突变不会直接影响微分项。

```c
// 位置式PID中，用测量值代替误差做微分
float derivative = measurement - pid->last_measurement; // 注意符号为负
pid->output += pid->Kd * (-derivative);  // 微分先行
pid->last_measurement = measurement;
```

### 3.5.3 微分噪声放大

**现象**：编码器量化误差 ±1 个脉冲，在 1ms 内变化，微分输出剧烈抖动，电机发出刺耳的"嗡嗡"声。

**防御——微分低通滤波**：

```c
#define D_FILTER_ALPHA 0.1f  // 滤波系数 0~1，越小越平滑

float derivative_raw = pid->error - pid->last_error;
pid->derivative_filtered = D_FILTER_ALPHA * derivative_raw 
                         + (1.0f - D_FILTER_ALPHA) * pid->derivative_filtered;
```

或者更简单地，**限制微分输出**：

```c
if (fabs(derivative) > DERIVATIVE_MAX) 
    derivative = (derivative > 0) ? DERIVATIVE_MAX : -DERIVATIVE_MAX;
```

> **嵌入式建议**：如果传感器噪声大（如廉价编码器、ADC 分辨率低），**宁可不用 D，也不要让系统抖。** 先用 PI 把系统调稳定，再尝试引入 D。

## 3.6 实时性与数值稳定性

### 3.6.1 中断 vs 主循环

| 方式 | 优点 | 缺点 | 推荐场景 |
|------|------|------|---------|
| **定时器中断** | 周期严格固定，实时性高 | 中断内不能执行太久 | 电机控制、姿态环 |
| **主循环 + 延时** | 简单，易于调试 | 周期不固定，受其他任务影响 | 温度控制、慢系统 |
| **RTOS 任务** | 可管理多个控制环 | 任务切换开销，抖动 | 多轴机械臂、复杂系统 |

**中断内的黄金法则**：

- PID 计算本身 < 100 条指令，微秒级，没问题。
- **不要在中断里做浮点 `printf`、串口发送、文件操作。**
- 如果必须用 RTOS，把 PID 放在高优先级任务，确保周期抖动 < 5%。

### 3.6.2 浮点数 vs 定点数

| 类型 | 优点 | 缺点 | 推荐平台 |
|------|------|------|---------|
| **float（32位）** | 直观，范围大，开发快 | Cortex-M0/M3 上慢，无 FPU 时耗时 | Cortex-M4F/M7（有硬件FPU） |
| **定点数（Q15/Q31）** | 无 FPU 时极快，确定性高 | 代码复杂，易溢出，调试难 | Cortex-M0/M3，DSP |
| **double** | 精度极高 | 在 32 位 MCU 上极慢，避免使用 | 不推荐 |

> **实践建议**：在 STM32F103（Cortex-M3，无 FPU）上，浮点 PID 单次计算约 50~100 µs，对 1kHz 控制环来说勉强够用。如果资源紧张，可以把 $K_p, K_i, K_d$ 放大成整数（如乘 1024），用整数运算，最后右移 10 位。

### 3.6.3 数值溢出

积分项是累加器，长时间运行可能溢出：

```c
// 防御：定期或在初始化时检查
pid->integral = 0.0f;  // 系统闲置时清零
```

或者使用**饱和积分**（始终限制在合理范围内）。

## 3.7 本章小结

| 要点 | 核心内容 |
|------|---------|
| **离散化** | 积分→累加，微分→差分，采样周期 $T$ 必须固定 |
| **位置式** | 输出绝对值，直观，需显式抗积分饱和 |
| **增量式** | 输出增量，故障安全，优先用于电机/调速 |
| **采样周期** | 控制环频率 ≥ 10× 系统带宽，电机通常 1~5 ms |
| **抗饱和** | 积分限幅、积分分离、条件积分（Clamping） |
| **微分防御** | 微分先行防设定值冲击，低通滤波防噪声 |
| **实时性** | 定时器中断最佳，中断内只做计算，不做 IO |
